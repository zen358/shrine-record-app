# 参拝記録アプリ（仮）

## サービス概要
- 参拝した神社の写真・メモ・位置情報・参拝日などを一元管理できる「デジタル参拝記録」サービスです。  
- スマホやPCから、神社巡りの記録を簡単に保存・整理できます。  
- 紙の御朱印帳では管理しきれない情報をまとめ、いつでも振り返れる参拝記録を提供します。

---

## このサービスへの思い・作りたい理由
神社巡りや御朱印集めをしていると、写真・メモ・地図・参拝日がバラバラに保存されてしまい、  
「この御朱印どこでもらったっけ？」「いつ参拝した？」ということがよく起こります。

私自身も、参拝した写真はスマホに残るものの、  
神社の名前や参拝した理由、お願いごと、御祭神などを整理して管理できず、後から振り返るのが難しいと感じていました。

そこで、  
**「参拝体験を1つにまとめて残せるアプリがあれば最高なのでは？」**  
という思いがきっかけで本サービスの開発を決めました。

将来的には参拝記録の共有やランキング、ベストショット写真の投稿など、  
神社巡りがさらに楽しくなるようなコミュニティ性も持たせたいと考えています。

---

## ユーザー層について
### 1. 神社巡り・御朱印集めが好きな一般ユーザー
- 神社巡りを趣味にしている人は記録ニーズが高く、継続利用が見込めるため。

### 2. 旅行・出張時にふらっと神社へ立ち寄るライトユーザー
- 毎回「この神社なんて名前だっけ？」が発生しがちで、デジタル記録との相性が良い。

### 3. 紙の御朱印帳に加えて、デジタルでも記録したいユーザー
- 紙だけでは記録しきれない情報（写真・位置情報・メモ）を補完できるため。

---

## サービスの利用イメージ
1. 神社で御朱印をいただき、スマホで写真を撮る。  
2. アプリを開き、  
   - 神社名  
   - 御祭神  
   - 参拝日  
   - お願いごと  
   - メモ  
   - 写真（スマホのカメラを起動して撮影 → そのままアップロード可能）  
   - 位置情報（自動 or 手動）  
   を登録する。  
3. 一覧画面で記録を振り返り、旅行の思い出や参拝の意味を再確認できる。  
4. （将来的に）記録を共有したり、他のユーザーの参拝ログを参考にできる。

→ 参拝体験を「記録・整理・共有」し、さらに神社巡りが楽しくなる価値を提供します。

## ユーザーの獲得について
- **SNS（X・Instagram）での発信**  
  御朱印・神社巡りに関心がある層が多く、写真映えのコンテンツと相性が良い。
- **神社巡りブログ・地域情報ブログと連携**  
  アプリの紹介記事を掲載してユーザーを誘導。
- **観光エリアの神社情報マップとのタイアップ（将来）**  
  神社巡りアプリとしての認知拡大を狙う。
- **既存御朱印アプリとの差別化を活かしたPR**  
  「記録が一番簡単に残せる」点を強調。

---

## サービスの差別化ポイント・推しポイント
### 1. 記録のしやすさに特化
- 他の御朱印系アプリは“検索”に強いものが多いが、  
  本サービスは“記録の簡単さ”を最優先に設計。

### 2. 写真・位置情報・メモを一画面で管理できる
- 参拝の情報が「写真アプリ」「メモ帳」「地図」に分散しない。

### 3. スマホカメラと連携したスムーズな写真登録
- スマホブラウザで開いた場合、写真選択ボタンをタップすると  
  そのまま **カメラが起動して撮影 → 即アップロード** が可能（`capture` 属性を利用）。

### 4. ユーザーネームを使った“記録の共有”を前提に設計
- 最初から「公開名」を設定することで、MVP後の共有機能にスムーズに移行可能。

### 5. 将来的なコミュニティ化を見据えたベース設計
- ベストショット写真ランキング  
- Thanks（いいね）機能  
- 人気参拝ルート  

など、神社巡りをもっと楽しくする方向性がはっきりしている。

## 使用する技術スタック
### フレームワーク
- Ruby on Rails 7.x  
- ERB / Turbo / Stimulus を利用したシンプルな構成

### データベース
- PostgreSQL（予定）

### デプロイ先（予定案）
- Render / Railway / Fly.io / Vercel（Rails対応PaaSから選択予定）

### 使用予定のライブラリ・機能
- **Devise**（認証）
- **Active Storage**（写真アップロード）
- **geocoder / 位置情報API（検討）**  
- **Google Maps API（将来の地図表示用）**
- **Rspec**（テスト）

### 一般的なCRUD以外の実装イメージ
- **写真アップロード**  
  → ActiveStorage + S3（本番環境）  
  → スマホでは `<input type="file" capture accept="image/*">` を利用し、  
     カメラから直接撮影 → 即アップロードに対応  
- **位置情報（緯度・経度）保存**  
  → geocoder で取得 or 端末の位置情報API利用  
- **参拝ルート作成機能（将来）**  
  → Google Directions API の利用を想定  
- **ランキング機能**  
  → 集計テーブルを定期バッチ更新 or SQLで集計
